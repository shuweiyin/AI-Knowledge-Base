# 🔵 工作室一班 — 在线课程介绍

> *面向12岁及以上学生的六周项目制课程。学生将制作并发布一款真正的音乐网页应用——部署到互联网上，拥有真实可访问的网址。*

---

## 📢 致家长 — 为什么这门课程很重要

"会用技术"和"能做出技术产品"之间，有一道巨大的鸿沟。工作室一班，就是帮助您的孩子跨越这道鸿沟的地方。

在这六节课里，您的孩子不会照着现成的教程一步步复制——他们将亲自规划、设计、构建、调试并发布一款完整的网页应用：一款名为 **"Beat Lab"** 的专业级音乐采样器，使用的是真正的软件开发工具和工作流程。

**这不是一个练习项目。** 完成后，他们的应用将拥有一个真实的互联网网址。可以发给朋友，放进学校的学习档案，甚至展示给未来的雇主。它可以在手机和电脑上运行，播放真实的音频，而且每一行代码都是他们自己做出来的。

**孩子将学到什么？**

| 技能 | 为什么重要 |
|---|---|
| **React** — 全球使用最广泛的网页开发框架 | Facebook、Instagram、Airbnb 等数百万网站都在使用 |
| **将项目发布到互联网** | 让代码从"只存在于我的电脑上"变成"全世界都能访问" |
| **音频编程** | 了解网页应用如何加载、触发和控制声音 |
| **专业界面设计** | 学会用 AI 生成精美的、有质感的用户界面 |
| **GitHub 版本管理** | 行业标准工具，用于保存、分享和协作开发代码 |
| **提示词工程** | 写出精准的 AI 指令，快速构建复杂功能 |
| **调试（Debugging）** | 阅读报错信息、找出问题并修复——开发者的核心技能 |
| **AI 伦理** | 思考作者署名权、采样授权和 AI 在创作中的角色 |

**孩子学完后将收获什么？**
- 一款部署在互联网上、拥有 **真实公开网址** 的音乐网页应用
- 一个 **GitHub 代码仓库** —— 专业作品集的起点
- 真实的**开发工作流程**体验：构建 → 测试 → 修复 → 发布
- 有底气说出：*"这是我做的。"*

## 📅 逐节课程安排

### 第一节 — 环境搭建与第一个项目 *（1小时）*

> **本节目标：** 每位学生在这节课结束时，都能在自己的屏幕上看到一个正在运行的 React 应用。

| | |
|---|---|
| **本节内容** | 全班一起安装 Node.js 和 npm（老师逐步带领每位学生完成）· 创建免费的 GitHub 和 Vercel 账号 · 创建一个全新的 React + Vite 项目 · 安装 Tailwind CSS 和 Howler.js · 在浏览器中运行应用并确认一切正常 · 介绍"Beat Lab"完成后的样子 |
| **核心概念** | 什么是 React · 什么是 npm · 什么是开发环境 · GitHub 和 Vercel 的作用 |
| **使用工具** | Node.js · npm · GitHub · Vercel · 终端（Terminal）· *（Cursor 可选，用于浏览文件）* |
| **学生收获** | "我的电脑已经配置得像一个真正的开发者，我能看到自己的应用在运行。" |

> 💡 **第一节课前无需任何准备。** Node.js 的安装将在第一节课开始时由老师带领全班一起完成。只需带上一台能联网的笔记本电脑即可。

---

### 第二节 — 设计采样器界面 *（1小时）*

> **本节目标：** 一个专业设计的采样器界面出现在屏幕上，学生能理解其组件结构。

| | |
|---|---|
| **本节内容** | 介绍 React 组件和 JSX（React 用来描述页面上所有内容的语言）· 使用 Claude Code 的 `frontend-design` 技能，为采样器生成一套大胆、专业的工作室风格界面 · 全班一起审阅生成的设计 · 个性化调整（颜色、布局、字体）· 理解 `SamplerPad` 组件与主 `App` 布局之间的关系 |
| **核心概念** | React 组件 · JSX · Props（属性）· 界面设计原则 · 什么样的界面让人感觉专业 |
| **使用工具** | Claude Code · React · Tailwind CSS |
| **学生收获** | "我有了一个真正好看的应用设计，而且我理解它为什么要这样构建。" |

---

### 第三节 — 声音与采样板 *（1小时）*

> **本节目标：** 采样器加载了8个真实音频样本，每个采样板都正确显示名称、按键和颜色。

|          |                                                                                                                          |
| -------- | ------------------------------------------------------------------------------------------------------------------------ |
| **本节内容** | 音频样本（底鼓、军鼓、踩镲、拍手声、808 低音等）· 下载并规范重命名文件 · 将文件加入项目的资源文件夹 · 构建 `sounds.js` 配置文件，定义每个采样板的属性 · 构建带有发光动画和按键标签的 `SamplerPad` 组件 |
| **核心概念** | 免版税音频 · 文件组织管理 · React 组件 Props · CSS 动画 · 配置文件                                                                          |
| **使用工具** | Claude Code · React                                                                                                      |
| **学生收获** | "我的应用有了8个加载了真实声音的采样板，每一个都长得恰好是我想要的样子。"                                                                                   |

---

### 第四节 — 让它发声 *（1小时）*

> **本节目标：** 点击采样板或按下对应键盘按键时，每个板子都会播放真实声音。

| | |
|---|---|
| **本节内容** | 将 Howler.js 与声音配置文件连接，让每个采样板播放对应的音频文件 · 添加键盘快捷键（A–K 键触发第1–8个采样板）· 构建主音量滑块 · 追踪哪个采样板正在播放，并显示发光状态 · 用点击和键盘逐一测试所有采样板 |
| **核心概念** | Howler.js · React 状态（`useState`）· `useEffect` 与事件监听器 · 键盘事件 · 为什么浏览器在用户交互前无法自动播放音频 |
| **使用工具** | Claude Code · Howler.js · React |
| **学生收获** | "每个采样板都能播放真实的声音——我可以只用键盘就打出一段节拍。" |

---

### 第五节 — 打磨细节 *（1小时）*

> **本节目标：** 应用看起来完整、专业，已准备好公开发布。

| | |
|---|---|
| **本节内容** | 在采样板下方添加"最近播放"指示器 · 为新用户添加键盘操作提示 · 确认采样板在手机上也能响应触摸操作 · 为每个采样板添加独立静音开关 · 运行 `npm run build`——最终构建检查 · 修复出现的所有报错和警告 · 全班一起进行最终视觉审查 |
| **核心概念** | 界面打磨与用户体验 · 移动端适配 · 构建流程 · 阅读并修复构建报错 |
| **使用工具** | Claude Code · React · 终端（Terminal） |
| **学生收获** | "我的应用通过了构建检查，在电脑和手机上看起来都很专业。" |

---

### 第六节 — 发布日 🚀 *（1小时）*

> **本节目标：** 应用在互联网上正式上线，拥有真实的公开网址，学生已经分享给他人。

| | |
|---|---|
| **本节内容** | 初始化 Git 仓库并提交所有代码 · 推送到 GitHub · 将 GitHub 仓库连接到 Vercel · 部署——看着构建日志完成 · 访问真实网址 · 将链接分享给班级 · 伦理讨论：AI 辅助写出的代码，作者是谁？免版税采样真的完全"免费"吗？· 每位学生简短展示自己的采样器 |
| **核心概念** | Git 提交 · GitHub 推送 · Vercel 部署 · 构建日志 · AI 著作权 · 音乐中的知识产权 |
| **使用工具** | Git · GitHub · Vercel · Claude Code |
| **学生收获** | "我的应用已经上线。我有了一个可以分享给全世界任何人的真实网址。" |

## 🎓 入学要求

### 推荐路径
建议先完成 [[Foundation-Level-Lesson-CN|基础班]]。基础班帮助学生建立信心、掌握核心词汇和提示词工程本能，能让工作室班的学习轻松许多，也更有成就感。

### 直接入学（经老师评估）
未完成基础班的学生同样欢迎申请。老师会进行一次简短的对话，确认学生是否具备足够基础。评估内容包括：

| 考察方向 | 具体内容 |
|---|---|
| **编程概念** | 能解释 if/else 是什么吗？什么是循环？能读懂简单的逻辑并预测结果吗？ |
| **AI 概念** | 什么是 AI？什么是大数据？机器是如何从案例中学习的？ |
| **提示词能力** | 是否用过 AI 创作过什么——图片、计划、文章？能写出清晰具体的提示词来实现目标吗？ |

> 评估没有"错误答案"。目的是了解学生目前的基础，让老师从第一天起就能给予最合适的支持。
